# service-overrides.json

소상공인시장진흥공단 상가정보 API(`scripts/fetchStores.js`)가 **채울 수 없는 편의정보**
— 주차, 와이파이, 콘센트, 드라이브스루, 테라스, 반려동물 동반 — 를 사람이 확인해 입력하는 파일입니다.
매장의 `id`(예: `투썸플레이스_MA0101...`)를 키로 사용합니다.

## 편의정보 3-state

편의정보는 **있음/없음/정보없음** 3단계로 관리합니다. "없음"과 "미확인"을 뭉개지 않습니다.

| 값 | 의미 | 앱 표시 |
|---|---|---|
| `true` | 있음(가능) | 배지 노출 |
| `false` | 없음(불가) | 배지 숨김 |
| `null` 또는 미기재 | 정보 없음 | "정보 없음" |

API로 받은 실데이터는 편의정보가 전부 `null`(정보 없음)로 시작합니다. 이 파일에서 확인된 값만
`true`/`false`로 덮어씁니다.

## 반영 방법

- `node scripts/applyOverrides.js` — API 재호출 없이 이 파일 값을 `카페 행사/stores/*.json`에 즉시 병합.
- `node scripts/fetchStores.js` — 전국 매장을 API로 다시 받으면서 이 파일 값을 자동 병합.

## 형식 예시

```json
{
  "투썸플레이스_MA0101...": {
    "_source": "출처 메모(뉴스/직접 확인 등)",
    "services": { "parking": true, "wifi": true, "outlet": true, "driveThrough": false, "terrace": false, "pet": false },
    "petPolicyType": "불가",
    "openingHours": "08:00 - 22:00",
    "phone": "02-123-4567"
  }
}
```

- `services` 키: `parking`(주차), `wifi`(와이파이), `outlet`(콘센트), `driveThrough`(드라이브스루), `terrace`(테라스), `pet`(반려동물 동반).
- `petPolicyType`: `가능` / `불가` / `테라스만 가능` / `이동가방 필요` / `소형견만 가능` / `정보 없음`.
- `openingHours`, `phone`: API에 없는 값을 확인했을 때만 입력(없으면 앱은 "정보 없음"으로 정직하게 표시).
- `_source`: 근거 메모(선택). 병합 로직은 무시하고 사람이 출처를 추적하는 용도.

> 현재 파일의 항목은 파이프라인 검증용 **예시(미검증)** 입니다. 실제 매장 정보로 교체하세요.

## 대량 확보 아이디어

편의정보를 대량으로 채우려면 매장별로 **카카오/네이버 지역(장소) API**를 조회해
주차·영업시간 등을 받아 이 파일에 병합하는 스크립트를 추가하면 됩니다(후속 과제).
반려동물 동반 여부는 공개 API가 없어 매장 공지/제보 기반 큐레이션이 현실적입니다.
